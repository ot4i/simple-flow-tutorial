Create the Message Flow

Create an Application project to hold the new message flow

  1. Message flows are created in the Toolkit using a drag and drop editor. The actions of this editor are executed on a file which has a .msgflow extension. Saving your changes causes this file to be updated in your workspace. Before creating the message flow, you will need to create a project which will be used to store it. The Toolkit offers several different types of project which can be used to store message flows. Users who are familiar with older versions of App Connect Enterprise might be familiar with using Integration Projects for this purpose. Although these projects still exist, a better choice particularly if you are entirely new to App Connect Enterprise would be to use an Application project.
  2. As is common with Eclipse based tools, there are several handy shortcuts and alternate ways to make navigation as easy as possible; but all these options will take you to the same New Application wizard. For example, if you've just launched a new workspace for the first time you will see a Quick Starts page with a link to Start by creating an application. Regardless of your starting point, you will be able to use the menus at the top of the Toolkit and select File - New - Application
  3. When the wizard launches, provide an Application name such as ExampleApplication and click Finish.

Create the message flow

  1. In the Application Development view (normally in the top left corner of your Toolkit) you will see that the application has been created. Underneath ExampleApplication, click where it says (New...). If you can't see where this is, you can also go to the top menu and choose File - New - Message flow. When the wizard launches, ensure that the Container selection is set to ExampleApplication. Note that the term Container, is just another word for project. Provide a Message Flow name such as ExampleMessageFlow, then click Finish.
  2. ExampleMessageFlow.msgflow will open in an editor. The flow starts off empty, but down the left side of the flow editor you will see a palette with a large number of labelled drawers, each with its own icon. A message flow node is a processing step in a message flow. You can wire message flow nodes together to define the order of processing. Each message flow node receives a message, performs a set of actions, and typically passes on the message to the next node in the message flow.
  3. Expand the HTTP drawer, and drag and drop two message flow nodes - an HTTPInput and an HTTPReply - on to the canvas. You will see four terminals on the right side of the HTTPInput message flow node. Wire the two message flow nodes together by clicking on the second terminal down (labelled Out if you hover over it) on the HTTPInput and then click on the HTTPReply node. If you find it easier, you could right-click on the HTTPInput node and select Create connections and use the menu which pops up. Save the message flow using the keyboard shortcut Ctrl-S, or from the top menu File - Save

Edit the properties of the HTTPInput node

  1. You will now see a small red cross displayed on the HTTPInput node. If you hover over it you will see an error telling you there is an unset, yet mandatory, property. To fix this, right-click on the HTTPInput node and select Properties. Focus will shift to the Properties tab which is found below the picture of the message flow itself, a few tabs to the left of the Tutorial Steps view where you are reading this text! Fill out the Path suffix for URL property with the value /ExampleMessageFlow being careful to include the leading slash character. Save the message flow again and the error should be resolved and the red cross on the HTTPInput node should disappear.
  2. Congratulations, you have just created your first message flow. Move to the Prepare section of the tutorial and we will prepare to test the flow.

Prepare an Integration Server to test the message flow

Create an Integration Server

  1. Next we will test the message flow using a local integration server, which we can create from within the Toolkit. If you're using a Toolkit earlier than version 11.0.0.7, then you will need to open an App Connect Enterprise Command Console and start an integration server that way. This alternate option is detailed in the Knowledge Center, but we will progress in this tutorial assuming you're using our most up to date Toolkit capabilities.
  2. From the Integration Explorer view in the bottom left corner of the Toolkit, right-click the Integration Servers folder and select the option to Create a local integration server. Leave all the options with their default values and click Finish. The local Integration Server will be started, using a project in your Toolkit's workspace as the work directory. Click OK to dismiss the Success dialog.

Test the message flow using the local Integration Server

Testing using the Flow Exerciser

  1. The message flow you created should still be open in the main window. At the top of the flow canvas, click on the circular red record button (just to the right of the Flow Exerciser text) to start the Flow Exerciser. Agree to the confirmation dialog which warns that the message flow is owned by an Application, and after a few seconds the background of the flow canvas will change to a grey colour. Another confirmation dialog will appear with a Ready to record message status. Read through the information on the dialog and click Close.
  2. Back at the top of the flow, click on the Send Message icon to create an input message.
  3. The Send Message dialog will open with instructions for how to create an input message. Click the New button, type some text (such as the classic Hello world for example!) in the Main window and click Send.
  4. A Progress Information dialog will appear showing the message has been sent, and a reply received. The same data should be echoed back in the reply.
  5. Congratulations, you have just successfully tested your first message flow! Click on the Close button and a green line should show the path the message took between the HTTPInput and HTTPReply nodes. Use the Flow Exerciser buttons at the top of the message flow to either send more test data or to return the flow to editting mode.
